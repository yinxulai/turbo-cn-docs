# 发布包

使用正确的工具，从 monorepo 将包发布到“npm”可能是一种非常令人满意和流畅的体验。

如果您想将 monorepo 的一些工作区作为包发布到 `npm`，则应遵循此设置。如果你不需要发布到 `npm`，你应该使用 [internal package](/repo/docs/handbook/sharing-code/internal-packages) 来代替。它们更容易设置和使用。

## 工具

您需要设置一些工具才能使其正常工作：

首先，您需要一个 [打包器](/repo/docs/handbook/publishing-packages/bundling) 将您的代码转换为 [`CommonJS`](https://en.wikipedia.org/wiki/CommonJS)， `npm` 上最常用的格式。您还需要设置一个开发脚本，以便您可以在本地开发中的工作区上工作。

最后，您需要一个用于 [发布和版本控制]（/repo/docs/handbook/publishing-packages/versioning-and-publishing）的工具。这将处理你的 monorepo 的包版本 _和_ 发布到 npm。
